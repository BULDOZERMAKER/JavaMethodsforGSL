Java-Methoden: Eine vollständige Anleitung

Einführung in Java-Methoden

Methoden in Java sind wiederverwendbare Codeblöcke, die eine bestimmte Aufgabe ausführen. Sie helfen dabei, Programme modular und leserlich zu gestalten.

Grundlagen der Methoden

Struktur einer Methode

Eine Methode in Java besteht aus folgenden Bestandteilen:

  • Rückgabetyp: Gibt an, welchen Datentyp die Methode zurückgibt (z. B. int, void).
  • Name der Methode: Der Bezeichner, mit dem die Methode aufgerufen wird.
  • Parameter: Werte, die an die Methode übergeben werden.
  • Methodenkörper: Der Code, der ausgeführt wird, wenn die Methode aufgerufen wird.

Beispiel einer einfachen Methode

// Methode zur Berechnung der Summe von zwei Zahlen
public int addiere(int a, int b) {
    return a + b;
}

Arten von Methoden

1. Void-Methoden

Führen eine Aktion aus, geben aber keinen Wert zurück.

public void druckeNachricht() {
    System.out.println("Hallo, Welt!");
}

2. Methoden mit Rückgabewert

Geben ein Ergebnis zurück.

public int quadriere(int zahl) {
    return zahl * zahl;
}

3. Statische Methoden

Können ohne ein Objekt der Klasse aufgerufen werden.

public static double berechneKreisfläche(double radius) {
    return Math.PI * radius * radius;
}

4. Instanzmethoden

Benötigen ein Objekt der Klasse, um aufgerufen zu werden.

public String gibName() {
    return this.name;
}

Methoden in Pseudocode

In Pseudocode lassen sich Methoden ähnlich wie in Java darstellen. Der Unterschied liegt in der vereinfachten Notation.

Beispiel:

Funktion Addiere(a, b)
    Rückgabe a + b
Ende Funktion

Komplette Anwendung in Pseudocode:

Prozedur Main()
    Zahl1 ← 5
    Zahl2 ← 10
    Ergebnis ← Addiere(Zahl1, Zahl2)
    Ausgabe "Die Summe ist: " + Ergebnis
Ende Prozedur

Funktion Addiere(a, b)
    Rückgabe a + b
Ende Funktion

Best Practices für Methoden

  • Verwende beschreibende Namen: Namen wie berechneSumme sind aussagekräftiger als methode1.
  • Vermeide zu viele Parameter: Nutze Objekte, um Daten zu bündeln.
  • Modularisierung: Teile den Code in kleine, spezifische Methoden auf.

Zusammenhang zwischen Java und Pseudocode

Pseudocode dient als Planungshilfe vor der eigentlichen Implementierung in Java. Er erleichtert das Verständnis der Logik, bevor syntaktische Details hinzugefügt werden.
